**Docker容器内目录**：
```
/opt/wvp/static/static/
├── index.html        # 入口HTML文件
├── static/           # 静态资源目录
│   ├── css/          # CSS样式文件
│   ├── js/           # JavaScript文件
│   ├── images/       # 图片资源
│   └── ...
└── favicon.ico       # 网站图标
```

**Volume挂载配置**（`start-remote-docker.sh`）：
```bash
-v "${SCRIPT_DIR}/volumes/wvp/static:/opt/wvp/static" \
```

**挂载映射关系**：
- 宿主机：`/home/wvp/docker/volumes/wvp/static` 
- 容器内：`/opt/wvp/static`
- 前端文件路径：`/opt/wvp/static/static/`

### 6. Spring Boot配置中的路径

**应用配置文件**（`docker/wvp/wvp/application-docker.yml`）：
```yaml
web:
    static:
        path: ${WEB_STATIC_PATH:/opt/wvp/static}
```

**环境变量**：
- `WEB_STATIC_PATH`：可自定义静态资源路径（默认：`/opt/wvp/static`）
- Spring Boot会在此路径下查找 `static/` 子目录

**实际读取路径**：
- 配置路径：`/opt/wvp/static`
- 实际文件路径：`/opt/wvp/static/static/`
- Spring Boot会自动查找 `static/` 子目录

### 7. WebMvc配置类

**配置类位置**：
```
src/main/java/com/genersoft/iot/vmp/conf/WebMvcConfig.java
```

**配置逻辑**：
1. 优先使用外部目录：`${web.static.path}/static/`
2. 如果外部目录不存在，回退到JAR包内：`classpath:/static/`

### 开发流程

```
1. 源代码
   web/src/ → web/public/
   
2. 构建
   npm run build:prod
   
3. 构建输出
   src/main/resources/static/
   
4. 打包（传统方式）
   JAR包内: BOOT-INF/classes/static/
   
5. 部署（前后端分离方式）
   宿主机: /home/wvp/docker/volumes/wvp/static/static/
   容器内: /opt/wvp/static/static/
```

### 部署流程（前后端分离）

```
1. 本地构建
   cd web && npm run build:prod
   → src/main/resources/static/

2. 打包传输
   tar czf static.tar.gz static/
   → 传输到远程服务器

3. 解压部署
   tar xzf static.tar.gz -C /home/wvp/docker/volumes/wvp/static/
   → /home/wvp/docker/volumes/wvp/static/static/

4. Volume挂载
   宿主机: /home/wvp/docker/volumes/wvp/static
   → 容器内: /opt/wvp/static

5. Spring Boot读取
   /opt/wvp/static/static/
   → 通过HTTP服务提供
```

## 重要说明

### 目录层级说明

**为什么有两个static目录？**

1. **第一个static**：`volumes/wvp/static/` 
   - 这是Volume挂载的根目录
   - 用于存放所有WVP相关的静态资源

2. **第二个static**：`volumes/wvp/static/static/`
   - 这是Vue构建输出的目录名（由`vue.config.js`中的`assetsDir: "static"`配置）
   - 实际的前端文件存放在这里

**Spring Boot读取逻辑**：
- 配置路径：`/opt/wvp/static`
- Spring Boot会在配置路径下查找 `static/` 子目录
- 实际读取：`/opt/wvp/static/static/`

### 部署时的注意事项

1. **构建输出结构**：
   ```
   src/main/resources/static/
   ├── index.html
   └── static/          ← 这个目录需要保持
       ├── css/
       ├── js/
       └── ...
   ```

2. **部署到宿主机**：
   ```bash
   # 正确方式：保持目录结构
   tar xzf static.tar.gz -C /home/wvp/docker/volumes/wvp/static/
   # 结果：/home/wvp/docker/volumes/wvp/static/static/
   
   # 错误方式：不要解压到static子目录
   tar xzf static.tar.gz -C /home/wvp/docker/volumes/wvp/static/static/
   ```

3. **验证部署**：
   ```bash
   # 检查宿主机文件
   ls -la /home/wvp/docker/volumes/wvp/static/static/
   
   # 检查容器内文件
   docker exec polaris-wvp ls -la /opt/wvp/static/static/
   
   # 检查Spring Boot日志
   docker logs polaris-wvp | grep "静态资源"
   ```
