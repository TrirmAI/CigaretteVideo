## 配置建议

### Docker环境配置
如果使用Docker Compose部署，建议在 `docker-compose.yml` 或 `docker-compose-remote.yml` 中配置：

```yaml
environment:
  DATABASE_HOST: 127.0.0.1  # 或远程数据库IP
  DATABASE_PORT: 3306
  DATABASE_USER: wvp_user
  DATABASE_PASSWORD: wvp_password
```

### 远程数据库配置
如果使用远程数据库，需要：

1. **确保数据库允许远程连接**
   ```sql
   -- 在MySQL中执行
   CREATE USER IF NOT EXISTS 'wvp_user'@'%' IDENTIFIED BY 'wvp_password';
   GRANT ALL PRIVILEGES ON wvp.* TO 'wvp_user'@'%';
   FLUSH PRIVILEGES;
   ```

2. **检查防火墙设置**
   - 确保远程服务器的3306端口已开放
   - 检查安全组规则（如果使用云服务器）

3. **修改配置文件**
   - 在 `application-docker.yml` 中通过环境变量设置 `DATABASE_HOST` 为远程IP

## 使用方法

### 检查本地数据库
```bash
cd docker
./check-database.sh
```

### 检查远程数据库
```bash
cd docker
./check-database.sh -H <远程IP> -P 3306 -U wvp_user -p wvp_password -D wvp
```

### 使用环境变量
```bash
export DATABASE_HOST=192.168.1.100
export DATABASE_PORT=3306
export DATABASE_USER=wvp_user
export DATABASE_PASSWORD=wvp_password
export DATABASE_NAME=wvp
cd docker
./check-database.sh
```

## 常见问题排查

### 1. 连接失败
- **检查数据库服务是否运行**: `docker ps | grep mysql`
- **检查端口是否开放**: `nc -zv <HOST> 3306`
- **检查用户名密码是否正确**
- **检查用户是否有远程连接权限**

### 2. 数据库不存在
- 运行初始化脚本: `数据库/2.7.4/初始化-mysql-2.7.4.sql`
- 或使用Docker Compose自动初始化

### 3. 权限不足
- 确保用户有足够的权限访问数据库
- 检查用户的主机权限（% 表示允许所有主机）
