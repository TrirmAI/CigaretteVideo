# WVP-Pro 前后端分离热更新配置说明

## 概述

本配置实现了前后端分离部署，**更新宿主机的前端文件后自动生效，无需重新编译Docker镜像和重启容器**。

## 实现原理

### 1. Volume挂载

通过Docker Volume将宿主机目录挂载到容器内：

```
宿主机: /home/wvp/docker/volumes/wvp/static
   ↓ (Volume挂载)
容器内: /opt/wvp/static
```

### 2. Spring Boot静态资源配置

通过 `WebMvcConfig.java` 配置类，Spring Boot会：
1. **优先从外部目录读取**：`file:/opt/wvp/static/static/`
2. **后备使用JAR包内资源**：`classpath:/static/`

### 3. 热更新机制

- Spring Boot使用 `file:` 协议读取外部目录
- 文件系统变化会**自动反映**到Spring Boot的资源加载中
- **无需重启容器**，更新文件后立即生效

## 目录映射关系

```
宿主机目录结构:
/home/wvp/docker/volumes/wvp/static/
└── static/                    ← 前端文件目录
    ├── index.html
    ├── favicon.ico
    └── static/
        ├── css/
        ├── js/
        └── ...

容器内目录结构:
/opt/wvp/static/
└── static/                    ← 通过Volume挂载，与宿主机同步
    ├── index.html
    ├── favicon.ico
    └── static/
        ├── css/
        ├── js/
        └── ...

Spring Boot读取路径:
/opt/wvp/static/static/        ← 配置的静态资源路径
```

## 配置说明

### 1. Docker Volume挂载

在 `start-remote-docker.sh` 中已配置：

```bash
-v "${SCRIPT_DIR}/volumes/wvp/static:/opt/wvp/static" \
```

### 2. Spring Boot配置

在 `application-docker.yml` 中：

```yaml
web:
    static:
        path: ${WEB_STATIC_PATH:/opt/wvp/static/static}
```

默认路径：`/opt/wvp/static/static`（容器内路径）

### 3. WebMvc配置类

`WebMvcConfig.java` 会自动检测外部目录，如果存在则优先使用。

## 使用方法

### 首次部署

1. **构建前端代码**
   ```bash
   cd web
   npm run build:prod
   ```

2. **部署到宿主机**
   ```bash
   cd docker
   ./deploy-frontend.sh
   ```

### 更新前端文件（热更新）

**方式1：使用部署脚本**
```bash
cd docker
./deploy-frontend.sh
```

**方式2：手动更新**
```bash
# 1. 本地构建
cd web && npm run build:prod

# 2. 打包
cd ../src/main/resources
tar czf /tmp/static.tar.gz static/

# 3. 传输到远程
scp /tmp/static.tar.gz root@172.31.127.47:/tmp/

# 4. SSH到远程服务器
ssh root@172.31.127.47

# 5. 解压替换文件（无需重启容器）
cd /home/wvp/docker
tar xzf /tmp/static.tar.gz -C volumes/wvp/static/
chmod -R 755 volumes/wvp/static/static

# 6. 验证（文件已自动生效）
curl http://localhost:18978/ | head -5
```

**方式3：直接替换单个文件**
```bash
# 在远程服务器上
cd /home/wvp/docker/volumes/wvp/static/static

# 替换index.html
vi index.html  # 或使用其他编辑器

# 文件保存后立即生效，无需重启
```

## 验证热更新

### 测试步骤

1. **修改前端文件**
   ```bash
   ssh root@172.31.127.47
   cd /home/wvp/docker/volumes/wvp/static/static
   echo "<!-- 测试热更新 -->" >> index.html
   ```

2. **立即访问页面**
   ```bash
   curl http://172.31.127.47:18978/ | grep "测试热更新"
   ```

3. **应该能看到修改内容**（无需重启容器）

### 检查配置是否生效

```bash
# 检查Volume挂载
docker inspect polaris-wvp | grep -A 3 "volumes/wvp/static"

# 检查容器内文件
docker exec polaris-wvp ls -la /opt/wvp/static/static/

# 检查服务日志（查找静态资源配置）
docker logs polaris-wvp | grep -i "静态资源\|外部"
```

## 优势

1. **热更新**：更新文件后立即生效，无需重启容器
2. **无需编译**：不需要重新构建JAR包和Docker镜像
3. **快速迭代**：前端开发调试更方便
4. **向后兼容**：如果外部目录不存在，自动使用JAR包内资源

## 注意事项

1. **文件权限**：确保文件有正确的读取权限（755）
2. **目录结构**：保持 `static/static/` 的目录结构
3. **文件同步**：Volume挂载是双向的，容器内和宿主机文件同步
4. **首次部署**：首次部署需要确保目录和文件都存在

## 故障排查

### 问题1：更新文件后不生效

**检查**：
```bash
# 1. 检查文件是否更新
ls -lh /home/wvp/docker/volumes/wvp/static/static/index.html

# 2. 检查容器内文件是否同步
docker exec polaris-wvp ls -lh /opt/wvp/static/static/index.html

# 3. 检查Spring Boot是否使用外部目录
docker logs polaris-wvp | grep "使用外部静态资源目录"
```

**解决**：
- 确保Volume挂载正确
- 检查文件权限
- 清除浏览器缓存

### 问题2：Spring Boot仍使用JAR包内资源

**原因**：`WebMvcConfig.java` 类需要编译到JAR包中才能生效

**解决**：
1. 重新构建WVP项目（包含WebMvcConfig类）
2. 重新构建Docker镜像
3. 重启容器

**临时方案**：如果不想重新构建，可以：
- 直接修改宿主机文件
- 使用Nginx反向代理（将静态文件请求转发到宿主机目录）
