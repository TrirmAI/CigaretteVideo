# 资源加载问题总结

## 问题确认

通过详细分析，确认了问题的根本原因：

### 调用过程

1. **Spring Boot启动时**：
   - `WelcomePageHandlerMapping` 自动配置，注册根路径 `/` 的处理器
   - 只查找 `classpath:/static/index.html`（JAR包内）
   - `WebMvcConfig` 后配置，注册外部目录资源处理器

2. **请求处理时**：
   - 访问 `/` → `WelcomePageHandlerMapping` 先匹配 → 返回JAR包内的 `index.html`
   - `WebMvcConfig` 的资源处理器未执行（因为已经找到处理器）

### 证据

- **MD5对比**：外部目录和返回内容的MD5不同
- **文件内容对比**：外部目录引用 `app.c0e6802d.css`，返回内容引用 `app.eb4e57e3.css`
- **日志顺序**：`WelcomePageHandlerMapping` 先注册，`WebMvcConfig` 后注册

## 尝试的解决方案

### 方案1：配置文件方式
在`application-docker.yml`中添加：
```yaml
spring:
    web:
        resources:
            add-mappings: false
```
**结果**：`WelcomePageHandlerMapping`仍然注册，配置未生效

### 方案2：创建IndexController
创建`IndexController`来处理根路径`/`，优先级高于`WelcomePageHandlerMapping`。

**问题**：服务启动失败，Spring Security配置错误

## 当前状态

- ✅ 问题已分析清楚
- ✅ 创建了IndexController（但导致服务启动失败）
- ⚠️ 需要修复服务启动问题

## 下一步建议

1. **修复IndexController**：确保不影响Spring Security配置
2. **或者使用其他方法**：例如修改WebMvcConfig的优先级
3. **或者接受现状**：虽然使用JAR包内的文件，但静态资源（CSS、JS）可能仍从外部目录加载

## 注意事项

- `spring.web.resources.add-mappings: false` 配置可能导致Spring Security配置问题
- IndexController需要确保不影响Spring MVC的正常初始化
- 需要确保服务能正常启动才能验证方案是否有效
