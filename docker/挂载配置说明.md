# WVP容器挂载配置说明

## ✅ 挂载配置已成功

### 挂载点配置

容器内的 `/opt/wvp` 目录已成功挂载到宿主机的 `/home/wvp` 目录。

### 挂载详情

| 容器内路径 | 宿主机路径 | 说明 |
|-----------|-----------|------|
| `/opt/wvp` | `/home/wvp` | 主目录挂载（包含wvp.jar） |
| `/opt/wvp/config` | `/home/wvp/docker/wvp/config` | 配置文件目录 |
| `/opt/wvp/logs` | `/home/wvp/docker/logs/wvp` | 日志目录 |
| `/opt/wvp/static` | `/home/wvp/docker/volumes/wvp/static` | 静态资源目录 |
| `/opt/ylcx/wvp/` | `/home/wvp/docker/wvp/wvp/` | Spring Boot配置文件目录 |

### 关键文件位置

- **JAR文件**: `/home/wvp/wvp.jar` (宿主机) = `/opt/wvp/wvp.jar` (容器内)
- **配置文件**: `/home/wvp/docker/wvp/wvp/application-docker.yml` (宿主机) = `/opt/ylcx/wvp/application-docker.yml` (容器内)
- **日志文件**: `/home/wvp/docker/logs/wvp/` (宿主机) = `/opt/wvp/logs/` (容器内)

### 启动命令

```bash
docker run -d \
  --name polaris-wvp \
  --network media-net \
  --restart always \
  -p 18978:18978/tcp \
  -p 8116:8116/udp -p 8116:8116/tcp \
  -p 21078:21078/tcp -p 21078:21078/udp \
  -v /home/wvp:/opt/wvp \
  -v /home/wvp/docker/wvp/config:/opt/wvp/config \
  -v /home/wvp/docker/wvp/wvp/:/opt/ylcx/wvp/ \
  -v /home/wvp/docker/logs/wvp:/opt/wvp/logs \
  -v /home/wvp/docker/volumes/wvp/static:/opt/wvp/static \
  [环境变量配置...] \
  localhost/wvp-pro:latest \
  java -Xms512m -Xmx1024m \
    -XX:+HeapDumpOnOutOfMemoryError \
    -XX:HeapDumpPath=/opt/wvp/logs/ \
    -jar /opt/wvp/wvp.jar \
    --spring.config.location=file:/opt/ylcx/wvp/
```

### 注意事项

1. **JAR文件位置**: JAR文件必须存在于 `/home/wvp/wvp.jar`，否则容器无法启动
2. **目录权限**: 确保 `/home/wvp` 目录有适当的权限，容器可以读写
3. **子目录挂载**: 子目录挂载（如 `/opt/wvp/config`）会覆盖主目录挂载中的对应子目录
4. **数据持久化**: 所有写入 `/opt/wvp` 的数据都会保存到宿主机的 `/home/wvp` 目录

### 验证方法

```bash
# 检查挂载点
docker inspect polaris-wvp | grep -A 10 Mounts

# 检查宿主机目录
ls -la /home/wvp/

# 检查容器内目录
docker exec polaris-wvp ls -la /opt/wvp/

# 验证文件同步
echo "test" > /home/wvp/test.txt
docker exec polaris-wvp cat /opt/wvp/test.txt
```
