# WVP-Pro 前后端分离热更新配置说明

## 目标

实现**更新宿主机的前端文件后自动生效，无需重新编译Docker镜像和重启容器**。

## 实现方案

### 方案概述

通过 **Docker Volume挂载** + **Spring Boot外部静态资源配置**，实现前端文件的热更新。

### 1. Docker Volume挂载

**挂载配置**（`start-remote-docker.sh`）：
```bash
-v "${SCRIPT_DIR}/volumes/wvp/static:/opt/wvp/static" \
```

**映射关系**：
- 宿主机：`/home/wvp/docker/volumes/wvp/static`
- 容器内：`/opt/wvp/static`
- **双向同步**：宿主机和容器内文件实时同步

### 2. Spring Boot静态资源配置

**配置文件**（`application-docker.yml`）：
```yaml
web:
    static:
        path: ${WEB_STATIC_PATH:/opt/wvp/static/static}
```

**WebMvc配置类**（`WebMvcConfig.java`）：
- 优先从外部目录读取：`file:/opt/wvp/static/static/`
- 后备使用JAR包内资源：`classpath:/static/`
- 使用 `file:` 协议，文件系统变化自动反映

### 3. 目录结构

```
宿主机:
/home/wvp/docker/volumes/wvp/static/
└── static/                    ← 前端文件目录
    ├── index.html
    ├── favicon.ico
    └── static/
        ├── css/
        ├── js/
        └── ...

容器内（通过Volume挂载）:
/opt/wvp/static/
└── static/                    ← 与宿主机同步
    ├── index.html
    ├── favicon.ico
    └── static/
        ├── css/
        ├── js/
        └── ...

Spring Boot读取:
/opt/wvp/static/static/         ← 配置的静态资源路径
```

## 使用方法

### 首次部署

1. **构建前端代码**
   ```bash
   cd web
   npm run build:prod
   ```

2. **部署到宿主机**
   ```bash
   cd docker
   ./deploy-frontend.sh
   ```

3. **确保目录结构正确**
   ```bash
   # 在远程服务器上
   ls -la /home/wvp/docker/volumes/wvp/static/static/
   # 应该看到：index.html, favicon.ico, static/ 目录
   ```

### 更新前端文件（热更新）

**方式1：使用部署脚本**
```bash
cd docker
./deploy-frontend.sh
```

**方式2：手动更新**
```bash
# 在远程服务器上直接修改文件
ssh root@172.31.127.47
cd /home/wvp/docker/volumes/wvp/static/static

# 修改文件（例如修改index.html）
vi index.html

# 保存后立即生效，无需重启容器！
```

**方式3：替换单个文件**
```bash
# 传输新文件
scp new-file.js root@172.31.127.47:/home/wvp/docker/volumes/wvp/static/static/static/js/

# 文件立即生效，无需重启
```

- `WebMvcConfig.java` 类需要编译到JAR包中才能生效
- 当前的WVP镜像不包含这个配置类
- 重新构建后，Spring Boot会自动检测外部目录并使用
