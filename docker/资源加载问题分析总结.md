# 资源加载问题分析总结

## 问题确认

通过详细分析，确认了问题的根本原因：

### 调用过程

1. **Spring Boot启动时**：
   - `WelcomePageHandlerMapping` 自动配置，注册根路径 `/` 的处理器
   - 只查找 `classpath:/static/index.html`（JAR包内）
   - `WebMvcConfig` 后配置，注册外部目录资源处理器

2. **请求处理时**：
   - 访问 `/` → `WelcomePageHandlerMapping` 先匹配 → 返回JAR包内的 `index.html`
   - `WebMvcConfig` 的资源处理器未执行（因为已经找到处理器）

### 证据

- **MD5对比**：外部目录和返回内容的MD5不同
- **文件内容对比**：外部目录引用 `app.c0e6802d.css`，返回内容引用 `app.eb4e57e3.css`
- **日志顺序**：`WelcomePageHandlerMapping` 先注册，`WebMvcConfig` 后注册

## 解决方案

已在 `application-docker.yml` 中添加配置：

```yaml
spring:
    web:
        resources:
            add-mappings: false
```

这个配置会禁用Spring Boot的默认静态资源映射，包括 `WelcomePageHandlerMapping`，让 `WebMvcConfig` 的资源处理器处理所有请求。

## 修复后的预期行为

1. `WelcomePageHandlerMapping` 被禁用
2. 访问 `/` 时，由 `WebMvcConfig` 的资源处理器处理
3. 优先查找外部目录：`file:/opt/wvp/static/index.html`
4. 如果外部目录找不到，回退到JAR包内：`classpath:/static/index.html`

## 验证方法

修复后验证：
1. 修改外部目录的 `index.html`（添加注释）
2. 访问 `http://localhost:18978/`
3. 检查返回内容是否包含注释
4. 对比MD5，确认使用外部目录的文件

## 注意事项

- 配置修改后需要重启服务才能生效
- 确保外部目录的文件存在且可读
- 如果外部目录文件不存在，会自动回退到JAR包内的文件
