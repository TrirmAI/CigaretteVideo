# WVP前后端分离部署说明

## ✅ 前后端分离配置完成

### 架构说明

1. **前端代码**：独立构建，部署到 `/opt/wvp/static`（容器内）= `/home/wvp/static`（宿主机）
2. **后端服务**：Spring Boot应用，提供RESTful API和静态资源服务
3. **分离优势**：
   - 前端代码可以独立更新，无需重新构建JAR包
   - 支持前端热更新（无需重启容器）
   - 前后端可以独立开发和部署

### 目录结构

```
宿主机: /home/wvp/
├── static/              # 前端静态文件（挂载到容器 /opt/wvp/static）
│   ├── index.html
│   ├── favicon.ico
│   └── static/
│       ├── css/
│       ├── js/
│       └── ...
├── wvp.jar              # 后端JAR文件
├── config/              # 配置文件目录
└── logs/                # 日志目录

容器内: /opt/wvp/
├── static/              # 前端静态文件（从宿主机挂载）
├── wvp.jar              # 后端JAR文件
├── config/              # 配置文件目录
└── logs/                # 日志目录
```

### 配置修改

#### 1. 前端构建配置 (`web/vue.config.js`)

```javascript
outputDir: "dist",  // 构建输出到dist目录
assetsDir: "static",  // 静态资源目录
```

#### 2. Spring Boot配置 (`application-docker.yml`)

```yaml
web:
    static:
        path: ${WEB_STATIC_PATH:/opt/wvp/static}
```

#### 3. WebMvcConfig配置

```java
// 默认路径：/opt/wvp/static（容器内路径，对应宿主机：/home/wvp/static）
String defaultPath = "/opt/wvp/static";
```

### 部署步骤

#### 1. 构建前端代码

```bash
cd web/
npm install
npm run build:prod
```

构建输出：`web/dist/` 目录

#### 2. 部署前端文件到远程服务器

```bash
# 打包前端文件
tar czf /tmp/wvp-frontend.tar.gz -C web/dist .

# 上传到远程服务器
scp /tmp/wvp-frontend.tar.gz root@172.31.127.47:/tmp/

# 在远程服务器解压
ssh root@172.31.127.47
cd /home/wvp
mkdir -p static
tar xzf /tmp/wvp-frontend.tar.gz -C static
chmod -R 755 static
```

#### 3. 更新配置文件并重启服务

```bash
# 更新配置文件
scp docker/wvp/wvp/application-docker.yml root@172.31.127.47:/home/wvp/docker/wvp/wvp/

# 重启服务
docker restart polaris-wvp
```

### 验证方法

1. **检查前端文件**
   ```bash
   ls -la /home/wvp/static/
   docker exec polaris-wvp ls -la /opt/wvp/static/
   ```

2. **检查服务日志**
   ```bash
   docker logs polaris-wvp | grep "使用外部静态资源目录"
   ```

3. **测试前端访问**
   ```bash
   curl http://localhost:18978/
   curl http://localhost:18978/index.html
   ```

### 前端更新流程

1. **本地构建前端**
   ```bash
   cd web/
   npm run build:prod
   ```

2. **打包并上传**
   ```bash
   tar czf /tmp/wvp-frontend.tar.gz -C web/dist .
   scp /tmp/wvp-frontend.tar.gz root@172.31.127.47:/tmp/
   ```

3. **远程部署**
   ```bash
   ssh root@172.31.127.47
   cd /home/wvp
   rm -rf static/*
   tar xzf /tmp/wvp-frontend.tar.gz -C static
   chmod -R 755 static
   ```

4. **验证更新**
   - 前端文件更新后，Spring Boot会自动读取新文件（无需重启容器）
   - 刷新浏览器即可看到更新

### 注意事项

1. **文件权限**：确保 `/home/wvp/static` 目录有适当的读写权限
2. **路径配置**：确保 `application-docker.yml` 中的 `web.static.path` 配置正确
3. **挂载配置**：确保Docker容器挂载了 `/home/wvp:/opt/wvp`
4. **API代理**：前端API请求会直接访问后端服务（端口18978）
