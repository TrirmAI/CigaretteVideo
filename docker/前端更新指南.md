# WVP-Pro 前端代码手动更新指南

## 概述

本指南介绍如何手动更新 WVP-Pro 项目的前端代码。前端代码位于 `web/` 目录，使用 Vue.js 开发。

**重要说明**：WVP-Pro 的前端文件是打包在 Java JAR 文件中的（位于 `src/main/resources/static`），因此更新前端需要重新构建整个项目并重启 WVP 服务。

## 前置要求

1. **本地环境**
   - Node.js >= 8.9
   - npm >= 3.0.0
   - 已安装 `sshpass` 或 `expect`（用于远程操作）

2. **远程服务器**
   - 已部署 WVP-Pro 服务
   - Nginx 容器正常运行
   - 具有 SSH 访问权限

## 方法一：完整重新构建（推荐）

由于前端文件打包在 JAR 中，需要重新构建整个项目。

### 步骤

1. **本地构建前端**
   ```bash
   cd /Users/andyapple/Downloads/wvp-GB28181-pro/web
   npm install --registry=https://registry.npmmirror.com
   npm run build:prod
   ```
   构建完成后，文件会自动输出到 `src/main/resources/static/` 目录。

2. **重新构建WVP服务**
   ```bash
   cd /Users/andyapple/Downloads/wvp-GB28181-pro
   mvn clean package -Dmaven.test.skip=true
   ```

3. **构建Docker镜像并部署**
   ```bash
   cd docker
   docker build -t localhost/wvp-pro:latest -f wvp/Dockerfile ..
   ```

4. **传输镜像到远程服务器并重启**
   ```bash
   # 保存镜像
   docker save localhost/wvp-pro:latest | gzip > wvp-pro-latest.tar.gz
   
   # 传输到远程
   scp wvp-pro-latest.tar.gz root@172.31.127.47:/tmp/
   
   # 在远程服务器上加载并重启
   ssh root@172.31.127.47
   docker load < /tmp/wvp-pro-latest.tar.gz
   docker restart polaris-wvp
   ```

## 方法二：仅更新前端文件（快速，需要访问容器内部）

如果只需要快速更新前端文件而不重新构建整个项目，可以直接替换容器中的静态文件。

### 步骤1: 本地构建前端代码

```bash
# 进入项目根目录
cd /Users/andyapple/Downloads/wvp-GB28181-pro

# 进入web目录
cd web

# 安装依赖（首次或依赖更新时）
npm install --registry=https://registry.npmmirror.com

# 构建生产版本
npm run build:prod
```

构建完成后，文件会输出到 `src/main/resources/static/` 目录。

### 步骤2: 部署到远程服务器

#### 方式A：直接替换WVP容器中的静态文件（快速，临时方案）

**注意**：这种方式替换的文件在容器重启后会丢失，因为文件是打包在JAR中的。

```bash
# 1. 打包构建产物（从src/main/resources/static目录）
cd /Users/andyapple/Downloads/wvp-GB28181-pro/src/main/resources
tar czf /tmp/wvp-static.tar.gz static/

# 2. 传输到远程服务器
scp /tmp/wvp-static.tar.gz root@172.31.127.47:/tmp/

# 3. SSH到远程服务器并替换文件
ssh root@172.31.127.47

# 4. 在远程服务器上执行
cd /home/wvp/docker

# 查找WVP容器中静态文件的位置（通常在jar包内，需要解压）
# 由于jar包内的文件无法直接替换，需要：
# 1. 停止容器
docker stop polaris-wvp

# 2. 创建临时目录并解压jar
docker run --rm -v /tmp:/work localhost/wvp-pro:latest sh -c \
  "cd /work && unzip -q /opt/wvp/wvp.jar -d wvp-extracted && \
   rm -rf wvp-extracted/BOOT-INF/classes/static/* && \
   tar xzf wvp-static.tar.gz -C wvp-extracted/BOOT-INF/classes/ && \
   cd wvp-extracted && zip -q -r /work/wvp-new.jar ."

# 3. 替换jar文件
docker cp /tmp/wvp-new.jar polaris-wvp:/opt/wvp/wvp.jar

# 4. 启动容器
docker start polaris-wvp

# 清理临时文件
rm -rf /tmp/wvp-extracted /tmp/wvp-new.jar /tmp/wvp-static.tar.gz
```

**更简单的方法**：直接挂载静态文件目录（需要修改容器启动方式）

```bash
# 1. 打包静态文件
cd /Users/andyapple/Downloads/wvp-GB28181-pro/src/main/resources
tar czf /tmp/wvp-static.tar.gz static/

# 2. 传输到远程服务器
scp /tmp/wvp-static.tar.gz root@172.31.127.47:/home/wvp/docker/volumes/wvp-static/

# 3. 在远程服务器上解压
ssh root@172.31.127.47
cd /home/wvp/docker/volumes/wvp-static
tar xzf wvp-static.tar.gz
rm wvp-static.tar.gz

# 4. 重启WVP容器（如果配置了volume挂载）
docker restart polaris-wvp
```

#### 方式B：重新构建nginx镜像（完整更新）

```bash
# 1. 打包整个web目录
cd /Users/andyapple/Downloads/wvp-GB28181-pro
tar czf /tmp/wvp-web.tar.gz web/

# 2. 传输到远程服务器
scp /tmp/wvp-web.tar.gz root@172.31.127.47:/tmp/

# 3. SSH到远程服务器
ssh root@172.31.127.47

# 4. 在远程服务器上执行
cd /home/wvp/docker

# 解压web目录
mkdir -p /tmp/wvp-web-temp
tar xzf /tmp/wvp-web.tar.gz -C /tmp/wvp-web-temp

# 构建nginx镜像
cd docker/nginx
docker build -t polaris-nginx:latest -f Dockerfile ../../

# 停止并删除旧容器
docker stop polaris-nginx
docker rm polaris-nginx

# 启动新容器（根据实际配置调整）
docker run -d \
  --name polaris-nginx \
  --network media-net \
  -p 8080:8080 \
  -v /home/wvp/docker/nginx/templates:/etc/nginx/templates \
  polaris-nginx:latest

# 清理临时文件
rm -rf /tmp/wvp-web-temp
rm -f /tmp/wvp-web.tar.gz
```

## 常见问题

### 1. 构建失败

**问题**：`npm run build:prod` 失败

**解决**：
- 检查 Node.js 版本：`node -v`（需要 >= 8.9）
- 清理并重新安装依赖：
  ```bash
  rm -rf node_modules package-lock.json
  npm install --registry=https://registry.npmmirror.com
  ```

### 2. 文件替换失败

**问题**：docker exec 命令失败

**解决**：
- 检查容器是否运行：`docker ps | grep nginx`
- 检查容器名称是否正确
- 尝试重启容器：`docker restart polaris-nginx`

### 3. 前端页面无法访问

**问题**：更新后页面无法访问或显示404

**解决**：
- 检查nginx容器日志：`docker logs polaris-nginx`
- 检查文件是否正确复制：`docker exec polaris-nginx ls -la /opt/dist`
- 检查nginx配置：`docker exec polaris-nginx cat /etc/nginx/conf.d/default.conf`

### 4. 样式或JS文件404

**问题**：页面可以访问但样式或JS文件404

**解决**：
- 检查文件路径是否正确
- 清除浏览器缓存
- 检查nginx配置中的静态文件路径

## 注意事项

1. **备份**：更新前建议备份当前的前端文件
   ```bash
   docker exec polaris-nginx tar czf /tmp/backup-frontend.tar.gz -C /opt/dist .
   docker cp polaris-nginx:/tmp/backup-frontend.tar.gz /tmp/
   ```

2. **版本控制**：建议在更新前记录当前版本，便于回滚

3. **测试**：更新后务必测试主要功能是否正常

4. **缓存**：更新后可能需要清除浏览器缓存才能看到最新版本

## 回滚操作

如果需要回滚到之前的版本：

```bash
# 1. 恢复备份文件
docker cp /tmp/backup-frontend.tar.gz polaris-nginx:/tmp/
docker exec polaris-nginx sh -c 'cd /opt/dist && rm -rf * && tar xzf /tmp/backup-frontend.tar.gz && rm /tmp/backup-frontend.tar.gz'
```
