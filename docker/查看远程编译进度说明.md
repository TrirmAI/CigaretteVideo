# 查看远程编译进度说明

## 概述

本文档说明如何查看远程服务器上的 Docker 镜像编译进度。

## 方法一：使用检查脚本（推荐）

使用 `check-build-status.sh` 脚本快速查看构建状态：

```bash
cd docker
./check-build-status.sh
```

该脚本会显示：
- 构建进程状态（是否正在运行）
- 最近30-50行构建日志
- 已构建的 Docker 镜像列表

## 方法二：实时查看构建日志

使用 `watch-build-log.sh` 脚本实时查看构建日志：

```bash
cd docker
./watch-build-log.sh
```

该脚本会：
- 检查构建进程是否正在运行
- 实时显示构建日志（类似 `tail -f`）
- 按 `Ctrl+C` 退出

## 方法三：手动 SSH 查看

### 查看构建状态

```bash
ssh root@172.31.127.47 "cd /home/wvp/build && \
  if [ -f build.pid ]; then \
    PID=\$(cat build.pid); \
    if ps -p \$PID > /dev/null 2>&1; then \
      echo '构建进行中，PID:' \$PID; \
    else \
      echo '构建已完成或失败'; \
    fi; \
  else \
    echo '构建进程未找到'; \
  fi"
```

### 查看构建日志

```bash
# 查看最后50行日志
ssh root@172.31.127.47 "tail -50 /home/wvp/build/build.log"

# 实时查看日志（按 Ctrl+C 退出）
ssh root@172.31.127.47 "tail -f /home/wvp/build/build.log"
```

### 查看 Docker 镜像

```bash
ssh root@172.31.127.47 "docker images | grep -E 'wvp-pro|wvp-nginx'"
```

## 构建流程说明

1. **启动构建**：运行 `./build-on-remote.sh` 启动远程构建
   - 构建会在后台运行
   - 构建进程 PID 保存在 `/home/wvp/build/build.pid`
   - 构建日志保存在 `/home/wvp/build/build.log`

2. **查看进度**：使用上述方法查看构建进度

3. **构建完成**：
   - `build.pid` 文件会被自动删除
   - 构建日志保留在 `build.log` 中
   - 可以使用 `docker images` 查看构建的镜像

## 常见问题

### Q: 如何判断构建是否完成？

A: 检查 `build.pid` 文件是否存在：
- 如果文件存在且进程正在运行：构建进行中
- 如果文件不存在：构建已完成或失败
- 查看 `build.log` 的最后几行确认构建结果

### Q: 构建失败怎么办？

A: 
1. 查看完整的构建日志：`ssh root@172.31.127.47 "cat /home/wvp/build/build.log"`
2. 检查错误信息
3. 修复问题后重新运行构建脚本

### Q: 如何停止正在进行的构建？

A: 
```bash
ssh root@172.31.127.47 "cd /home/wvp/build && \
  if [ -f build.pid ]; then \
    PID=\$(cat build.pid); \
    kill \$PID; \
    rm -f build.pid; \
    echo '构建已停止'; \
  fi"
```

## 脚本说明

- **`build-on-remote.sh`**: 在远程服务器上启动构建任务（后台运行）
- **`check-build-status.sh`**: 检查构建状态和显示日志
- **`watch-build-log.sh`**: 实时查看构建日志

## 注意事项

1. 确保已安装 `sshpass` 或 `expect` 工具
2. 确保 SSH 连接正常
3. 构建过程可能需要较长时间（10-30分钟），请耐心等待
4. 构建日志会持续增长，建议定期清理旧日志
