# 资源加载调用过程分析

## 问题确认

通过MD5对比和文件内容对比，确认：
- **外部目录index.html**: 引用 `app.c0e6802d.css` 和 `app.0abbaba8.js`
- **实际返回的index.html**: 引用 `app.eb4e57e3.css` 和 `app.bdc45c77.js`
- **结论**: 系统使用的是JAR包内的文件，而不是外部目录的文件

## 调用过程分析

### 1. Spring Boot启动时的资源处理器注册顺序

```
启动顺序：
1. Spring Boot自动配置阶段
   ↓
2. WelcomePageHandlerMapping 自动配置
   - 查找 classpath:/static/index.html
   - 注册根路径 "/" 的处理器
   - 日志：Adding welcome page: class path resource [static/index.html]
   ↓
3. WebMvcConfig 配置加载
   - 注册外部目录资源处理器：file:/opt/wvp/static/
   - 注册JAR包内资源处理器：classpath:/static/
   - 日志：使用外部静态资源目录: /opt/wvp/static/
```

### 2. 请求处理流程

当访问 `http://localhost:18978/` 时：

```
请求到达
  ↓
DispatcherServlet 接收请求
  ↓
HandlerMapping 查找处理器
  ↓
WelcomePageHandlerMapping.match(request, "/")
  ✅ 匹配成功！优先级：最高
  ↓
返回 classpath:/static/index.html（JAR包内）
  ↓
WebMvcConfig 的资源处理器（未执行，因为已经找到处理器）
```

### 3. 为什么WebMvcConfig不生效？

**关键问题**：
1. **优先级**：`WelcomePageHandlerMapping` 的优先级高于 `ResourceHandlerRegistry`
2. **注册时机**：`WelcomePageHandlerMapping` 在 `WebMvcConfig` 之前注册
3. **路径匹配**：`WelcomePageHandlerMapping` 专门处理根路径 `/`，一旦匹配就不再查找其他处理器
4. **资源查找范围**：`WelcomePageHandlerMapping` **只查找** `classpath:` 资源，**不会查找** `file:` 协议的外部目录

### 4. 日志证据

```
2025-12-18 10:17:33.056 [main]  INFO --- o.s.b.a.web.servlet.WelcomePageHandlerMapping: 59 Adding welcome page: class path resource [static/index.html]
2025-12-18 10:17:33.145 [main]  INFO --- com.genersoft.iot.vmp.conf.WebMvcConfig: 45 使用外部静态资源目录: /opt/wvp/static/
```

**时间顺序**：
- `WelcomePageHandlerMapping` 先注册（10:17:33.056）
- `WebMvcConfig` 后注册（10:17:33.145）

## 解决方案

### 方案：禁用WelcomePageHandlerMapping

在`application-docker.yml`中添加配置，禁用Spring Boot的默认欢迎页面：

```yaml
spring:
    web:
        resources:
            add-mappings: false
```

**注意**：这个配置会禁用所有默认的静态资源映射，包括：
- WelcomePageHandlerMapping
- 默认的静态资源位置（classpath:/static/等）

但我们的`WebMvcConfig`已经配置了资源处理器，所以不会有影响。

## 修复后的调用流程

```
请求到达
  ↓
DispatcherServlet 接收请求
  ↓
HandlerMapping 查找处理器
  ↓
WelcomePageHandlerMapping.match(request, "/")
  ❌ 未找到（已禁用）
  ↓
WebMvcConfig 的资源处理器匹配
  ✅ 匹配成功：/**
  ↓
查找 file:/opt/wvp/static/index.html（外部目录）
  ✅ 找到！
  ↓
返回外部目录的index.html
```
