# WVP-Pro 本地开发启动指南

本指南将帮助您在本地启动 WVP-Pro 项目的前端和后台服务，同时使用 Docker 运行流媒体服务器和数据库。

## 📋 前置要求

1. **Java 21+** - 后端服务需要
2. **Node.js 8.9+** 和 **npm 3.0+** - 前端服务需要
3. **Maven** - 用于编译后端项目
4. **Docker** 和 **Docker Compose** - 用于运行数据库和流媒体服务器

## 🚀 快速启动

### 方式一：使用启动脚本（推荐）

```bash
# 在项目根目录执行
./start-local.sh
```

启动脚本会自动：
1. 检查环境依赖
2. 启动 Docker 服务（Redis、MySQL、流媒体服务器）
3. 编译后端项目（如果需要）
4. 启动后端服务
5. 启动前端服务

### 方式二：手动启动

#### 1. 启动 Docker 服务

```bash
cd docker
docker-compose -f docker-compose-local.yml up -d
```

这将启动：
- **Redis** (端口: 6379)
- **MySQL** (端口: 3306)
- **ZLMediaKit 流媒体服务器** (端口: 8080, 10935, 5540, 10000, 8001)

#### 2. 编译后端项目

```bash
# 在项目根目录
mvn clean package -DskipTests
```

#### 3. 启动后端服务

```bash
java -jar -Dspring.profiles.active=dev target/wvp-pro-*.jar
```

或者使用后台运行：

```bash
nohup java -jar -Dspring.profiles.active=dev target/wvp-pro-*.jar > logs/wvp-startup.log 2>&1 &
```

#### 4. 启动前端服务

```bash
cd web

# 首次运行需要安装依赖
npm install

# 启动开发服务器
npm run dev
```

## 📍 服务地址

启动成功后，可以通过以下地址访问：

- **前端界面**: http://localhost:9528
- **后端 API**: http://localhost:18080
- **API 文档**: http://localhost:18080/doc.html

## 🔧 配置文件说明

### 后端配置

配置文件位置: `application-dev.yml`

主要配置项：
- **Redis**: `127.0.0.1:6379` (无密码)
- **MySQL**: `127.0.0.1:3306` (用户名: root, 密码: root)
- **流媒体服务器**: `127.0.0.1:8080`
- **后端服务端口**: `18080`

### Docker 配置

Docker Compose 文件: `docker/docker-compose-local.yml`

包含的服务：
- `polaris-redis`: Redis 数据库
- `polaris-mysql`: MySQL 数据库
- `polaris-media`: ZLMediaKit 流媒体服务器

## 🛑 停止服务

### 停止 Docker 服务

```bash
cd docker
docker-compose -f docker-compose-local.yml down
```

### 停止后端服务

```bash
# 查找进程
ps aux | grep wvp-pro

# 停止进程
kill <PID>
```

### 停止前端服务

在运行前端服务的终端中按 `Ctrl+C`

## 📝 常见问题

### 1. 端口被占用

如果遇到端口被占用的问题：

```bash
# 检查端口占用
lsof -i :18080  # 后端端口
lsof -i :9528   # 前端端口
lsof -i :6379   # Redis端口
lsof -i :3306   # MySQL端口

# 或者使用
netstat -an | grep <端口号>
```

### 2. Docker 服务启动失败

检查 Docker 服务状态：

```bash
cd docker
docker-compose -f docker-compose-local.yml ps
docker-compose -f docker-compose-local.yml logs
```

### 3. 后端服务无法连接数据库

确保：
- Docker 服务已启动
- MySQL 容器正常运行
- `application-dev.yml` 中的数据库配置正确

检查 MySQL 连接：

```bash
docker exec -it docker-polaris-mysql-1 mysql -uroot -proot -e "SHOW DATABASES;"
```

### 4. 前端无法连接后端

确保：
- 后端服务已启动（端口 18080）
- 前端代理配置正确（`web/vue.config.js`）

检查后端服务：

```bash
curl http://localhost:18080/openapi/v1/health
```

### 5. 编译失败

确保：
- Java 版本为 21 或更高
- Maven 已正确安装
- 网络连接正常（需要下载依赖）

## 🔍 查看日志

### 后端日志

```bash
tail -f logs/wvp-startup.log
# 或
tail -f logs/wvp-*.log
```

### 前端日志

```bash
tail -f logs/frontend.log
```

### Docker 服务日志

```bash
cd docker
docker-compose -f docker-compose-local.yml logs -f
```

查看特定服务日志：

```bash
docker-compose -f docker-compose-local.yml logs -f polaris-mysql
docker-compose -f docker-compose-local.yml logs -f polaris-redis
docker-compose -f docker-compose-local.yml logs -f polaris-media
```

### 检查后端服务
