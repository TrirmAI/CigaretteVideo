# 拉流错误排查指南

## 问题现象

界面上显示拉流错误，控制台日志显示：
- `h265web.js` 相关日志
- `missile.js:1289 let's get the ball rolling!`
- `Getting token...`
- 流数量为0

## 问题分析

### 1. 根本原因

**当前没有流在运行**。WebRTC播放需要流已经存在，如果流不存在，播放会失败。

### 2. 播放流程

```
1. 用户点击播放
   ↓
2. WVP调用播放接口 /api/play/start/{deviceId}/{channelId}
   ↓
3. WVP向设备发送INVITE请求
   ↓
4. 设备开始推流到ZLMediaKit
   ↓
5. ZLMediaKit收到流并通知WVP
   ↓
6. WVP返回流信息（包含WebRTC地址）
   ↓
7. 前端播放器使用WebRTC地址播放
```

**问题出现在步骤4-5**：流还没有建立，但前端已经尝试播放。

## 排查步骤

### 步骤1：检查流是否存在

```bash
# 检查当前流列表
curl "http://127.0.0.1:8080/index/api/getMediaList?secret=5j76zqeppUp7mpsawXIGo5gLW1O6j7CR"
```

**如果返回空数组 `[]`**，说明没有流在运行。

### 步骤2：检查设备状态

1. 登录WVP管理界面
2. 进入"设备管理"
3. 检查设备是否在线
4. 检查通道是否存在

### 步骤3：检查播放接口响应

在浏览器控制台查看：
- 播放接口是否返回成功
- 返回的流信息中是否包含`rtc`字段
- `rtc`地址是否正确

### 步骤4：检查播放器选择

根据代码，播放器选择逻辑：
- **jessibuca**: 使用 `ws_flv` 或 `wss_flv`
- **webRTC**: 使用 `rtc` 或 `rtcs`
- **h265web**: 使用 `ws_flv` 或 `wss_flv`

如果选择了WebRTC播放器但`rtc`地址为空，会尝试使用`ws_flv`或`flv`。

## 解决方案

### 方案1：确保流已建立（推荐）

**步骤**：
1. 点击播放按钮
2. **等待3-5秒**让流建立
3. 如果播放失败，检查：
   - 设备是否在线
   - 通道是否存在
   - 网络是否正常

### 方案2：检查播放器选择

如果使用WebRTC播放器：
1. 确认返回的流信息中包含`rtc`字段
2. 确认`rtc`地址格式正确：`http://172.31.127.42:8080/index/api/webrtc?...`
3. 如果`rtc`为空，尝试切换到其他播放器（如jessibuca）

### 方案3：检查WebRTC配置

确认以下配置正确：

**WVP配置** (`application-dev.yml`):
```yaml
media:
  http-port: 8080
  stream-ip: 172.31.127.42
```

**ZLMediaKit配置** (`config.ini`):
```ini
[rtc]
port=8001
tcpPort=8001
externIP=172.31.127.42
```

### 方案4：查看详细错误日志

**浏览器控制台**：
- 查看是否有WebRTC相关错误
- 查看是否有网络请求失败
- 查看播放器事件回调

**WVP日志**：
```bash
# 查看WVP日志
tail -f logs/wvp-*.log | grep -i "play\|webrtc\|rtc"
```

**ZLMediaKit日志**：
```bash
# 查看Docker容器日志
docker logs -f docker-polaris-media-1 | grep -i "webrtc\|rtc\|stream"
```

## 常见错误及解决方法

### 错误1：流不存在

**现象**：
- 控制台显示"流不存在"
- WebRTC播放器返回错误码-400

**解决方法**：
1. 确认设备已在线
2. 确认通道存在
3. 等待流建立（3-5秒）
4. 检查ZLMediaKit是否收到流

### 错误2：ICE协商失败

**现象**：
- 控制台显示"ICE 协商出错"
- WebRTC连接失败

**解决方法**：
1. 检查RTC端口8001是否开放
2. 检查防火墙设置
3. 检查`externIP`配置是否正确
4. 尝试使用TCP模式（如果UDP不通）

### 错误3：offer/answer交换失败

**现象**：
- 控制台显示"offer anwser 交换失败"
- WebRTC地址无法访问

**解决方法**：
1. 检查WebRTC API地址是否正确
2. 检查HTTP端口8080是否可访问
3. 检查流是否已建立
4. 查看ZLMediaKit日志确认API调用

### 错误4：h265web播放器错误

**现象**：
- `h265web.js` 相关错误
- `missile.js` 初始化失败

**解决方法**：
1. 确认流格式支持H.265
2. 尝试切换到其他播放器（jessibuca或WebRTC）
3. 检查浏览器是否支持H.265解码
4. 确认流地址格式正确（ws_flv）
